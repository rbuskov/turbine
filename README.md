# Turbine

Turbine is an experimental ASP.NET Core library that eliminates the need for dedicated request/response DTO classes,
provides runtime request validation, and helps with accurate OpenAPI spec generation. This simpifies REST API
development significantly and prevents drift between model shapes, validation logic and published specification.

Turbine is built around a fluent API that allows you to build OpenAPI-style schemas from your domain models. Requests
and response payloads are represented as generic JsonElement objects internally, and the Turbine schemas handle the
conversion to and from your domain models with simple ToJson/FromJson methods.

Turbine also offers Accepts/Produces extension methods that add schemas and operations the OpenAPI specification
generated by ASP.NET Core at runtime, as well as a drop-in middleware component that will perform request validation
based on schemas used in your project.

Turbine builds on the ideas behind the [SpecGuard](https://github.com/rbuskov/SpecGuard) validation library and is work
in progress.
